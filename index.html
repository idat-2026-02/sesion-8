1. El Brief del Proyecto

¿Qué es?

Es el documento fundacional de cualquier proyecto web. Actúa como un puente de comunicación entre el cliente y el equipo de desarrollo/diseño.

Sin un brief claro, el proyecto carece de dirección y es propenso a constantes cambios ("scope creep").

Elementos Clave

  • 🎯 Objetivos del proyecto: ¿Qué queremos lograr?
  • 👥 Público Objetivo: ¿A quién nos dirigimos?
  • ⚙️ Requerimientos Técnicos: Integraciones, hosting.
  • 🎨 Guía de Estilo: Colores, tipografías, tono.
  • 📅 Cronograma e Hitos: Fechas de entrega.

2. Vínculos Avanzados

La etiqueta <a> no solo sirve para navegar entre páginas HTML. Cambiando el protocolo en el atributo href, podemos interactuar con el sistema operativo del usuario.

Correos (mailto)

Abre el cliente de correo predeterminado del usuario.

<a href="mailto:hola@idat.edu.pe">
  Enviar un correo
</a>

Teléfonos (tel)

Inicia una llamada (muy útil en dispositivos móviles).

<a href="tel:+51999888777">
  Llámanos ahora
</a>

Descarga de Archivos (download)

Añadiendo el atributo booleano download, forzamos la descarga del archivo en lugar de que el navegador intente abrirlo.

<a href="assets/silabo.pdf" download>Descargar Sílabo (PDF)</a>

3. DIV como Contenedor Genérico

El elemento <div> (División) es el contenedor a nivel de bloque más utilizado en HTML. No tiene ningún significado semántico por sí mismo.

  • Se utiliza puramente para agrupar otros elementos.
  • Ideal para aplicar estilos CSS (usando class o id).
  • Fundamental para crear layouts (estructuras de filas y columnas) con Flexbox o CSS Grid.
  • Nota: Siempre prioriza etiquetas semánticas (header, main, section) antes de usar un div.
<div class="tarjeta">

Título de la tarjeta

El div envuelve la imagen simulada, este título y este párrafo, permitiendo moverlos y estilizarlos como un solo bloque.

</div>

4. Formatos de Imagen Modernos

Elegir el formato correcto es crucial para la calidad visual y el rendimiento web.

📸

PNG / JPG

Los Clásicos

JPG: Ideal para fotografías. No soporta transparencia. Compresión con pérdida.

PNG: Soporta transparencia (canal Alpha). Compresión sin pérdida pero archivos más pesados.

⚡

WebP

El Estándar Moderno

Creado por Google. Ofrece compresión con y sin pérdida, y soporta transparencia.

Archivos hasta un 30% más pequeños que JPG/PNG con la misma calidad visual.

📐

SVG

Gráficos Vectoriales

Basado en código XML (matemáticas, no píxeles).

Escalabilidad infinita sin perder calidad. Perfecto para logotipos, iconos e ilustraciones planas.

5. Optimización de Recursos Visuales

Las imágenes suelen ser el recurso que más ancho de banda consume en una página web. La optimización es obligatoria para un buen UX y SEO.

1 Dimensiones Exactas

No cargues una imagen de 4000x3000px si solo la vas a mostrar en un recuadro de 400x300px. Redimensiona antes de subir.

2 Herramientas de Compresión

Usa herramientas como TinyPNG o Squoosh para reducir el peso del archivo sin que el ojo humano note la diferencia.

Carga Diferida (Lazy Loading)

Añade el atributo loading="lazy" a tus imágenes. Esto le dice al navegador que solo descargue la imagen cuando esté a punto de aparecer en la pantalla del usuario, ahorrando datos y tiempo de carga inicial.

<img 
  src="foto-optimizada.webp" 
  alt="Descripción SEO" 
  loading="lazy"
>
